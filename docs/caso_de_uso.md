# Caso de Uso Principal (Plantilla del Estudiante)

## Nombre del caso de uso
Ejemplo: “Generar plan de estudio diario”.

## Actor principal
¿Quién usa el sistema? (por ejemplo: estudiante de ingeniería).

## Precondiciones
¿Qué debe existir antes de ejecutar el caso? (archivos, permisos, datos básicos).

## Disparador
¿Qué acción inicia el caso de uso?

## Flujo principal
1. El usuario ejecuta el programa.
2. El sistema solicita o recibe datos.
3. El sistema procesa la información.
4. El sistema muestra resultado en terminal y/o archivo.

## Flujos alternos
- **A1:** Entrada inválida.
- **A2:** Archivo no encontrado.
- **A3:** Parámetro incompleto.

## Postcondiciones
¿Qué queda listo al finalizar? (resultado, archivo generado, estado actualizado).

## Criterios de aceptación
Define al menos 3 criterios verificables.
