# Plan de Pruebas (Plantilla del Estudiante)

## Objetivo del plan
Validar que la práctica cumple su caso de uso principal con pruebas simples y repetibles en entorno local.

## Alcance de pruebas
- Pruebas funcionales básicas de entrada/salida.
- Manejo de errores comunes.
- Verificación manual de resultados esperados.

## Casos de prueba mínimos

| ID | Escenario | Entrada | Resultado esperado | Estado |
|---|---|---|---|---|
| CP-01 | Flujo principal válido | Datos correctos | Salida correcta en terminal | Pendiente |
| CP-02 | Entrada inválida | Formato incorrecto | Mensaje de error claro | Pendiente |
| CP-03 | Archivo faltante (si aplica) | Ruta inexistente | Error controlado sin crash | Pendiente |

## Evidencia
Describe cómo documentarás evidencia:
- capturas de terminal,
- bitácora corta en `tests/test_plan.md`,
- comandos usados para ejecutar pruebas.

## Criterio de salida
La práctica se considera aceptable cuando al menos el flujo principal y un flujo alterno pasan con evidencia.
